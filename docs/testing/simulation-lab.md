# Facttic Simulation Lab

## Overview
The Simulation Lab is a specialized testing environment designed to stress-test the Facttic AI Governance system using synthetic adversarial traffic. It simulates various attack vectors and policy violations to ensure that the Detection Engine, Guardrail Engine, and Policy Resolver are functioning correctly.

## Architecture
Simulation traffic is strictly routed through the **Governance Control API** (`/api/governance/execute`). This ensures that synthetic events are processed by the same pipeline as production traffic and are recorded in the **Forensic Event Ledger**.

### Routing Path
1. **Simulator Execution**: `TrafficSimulator.runScenario` generates prompts based on the selected scenario.
2. **API Request**: Sends a `POST` request to `/api/governance/execute` with simulation metadata.
3. **Pipeline Processing**: The Governance Pipeline executes Signal Analysis, Interception, and Resolution.
4. **Ledger Persistence**: The forensic ledger records the event with a `simulation_id`.
5. **UI Updates**: The Simulation Dashboard and Governance Control Center update in real-time.

## Supported Scenarios

| Scenario | Description |
|----------|-------------|
| **Prompt Injection** | Adversarial system overrides and jailbreak attempts. |
| **Hallucination** | High-confidence false generation testing. |
| **Policy Violation** | Organization-specific PII and security rule breaches. |
| **Context Overflow** | Resource and memory stress testing. |
| **Toxic Output** | Harmful or biased content generation. |

## Metadata & Auditability
Every simulation run includes the following metadata fields:
- `simulation_id`: Unique UUID to correlate all events in a single run.
- `scenario_type`: The ID of the scenario being executed.
- `traffic_volume`: Number of prompts generated in the run.

## Forensic Integration
Events generated by the Simulation Lab are explicitly tagged with a `simulation_id` in the `governance_event_ledger`, allowing forensic investigators to filter out synthetic traffic from actual production incidents.

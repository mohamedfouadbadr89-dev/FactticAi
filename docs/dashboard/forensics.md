# Forensics Dashboard

## Overview
The Forensics Dashboard (`/dashboard/forensics`) is a diagnostic environment designed for deep-dive investigations of AI session failures, policy breaches, and behavioral anomalies. 

## Dashboard Layout
The dashboard is organized into three primary analytical columns:

1.  **Execution Timeline (Left)**: A vertical chronological ledger of governance events (e.g., policy violations, drift detections, escalations) extracted from the `conversation_timeline` table.
2.  **Intelligence Depth (Center)**:
    *   **Root Cause Analysis (RCA)**: Visualizes causal chains generated by the `RcaGraphEngine`. It identifies the primary failure point (e.g., Model Drift vs. Prompt Injection) with a calculated confidence score.
    *   **Behavioral Forensics**: Displays anomaly signals such as intent drift, instruction override attempts, and context saturation levels.
3.  **Conversational Context (Right)**: A high-fidelity replay of the session turns, allowing auditors to inspect the raw content and associated risk metadata for each exchange.

## APIs & Data Sources
- **Sessions**: `GET /api/governance/sessions` (filtered for high-risk activity).
- **RCA Graph**: `GET /api/forensics/rca-graph/[sessionId]` - Source: `RcaGraphEngine`.
- **Behavior**: `GET /api/forensics/behavior/[sessionId]` - Source: `BehaviorForensicsEngine`.
- **Timeline**: `GET /api/governance/timeline/[sessionId]` - Source: `conversation_timeline` table.
- **Turns**: `GET /api/governance/sessions/[sessionId]/turns` - Source: `session_turns` table.

## Signal Visualization
- **Red Signals**: Critical policy violations or detected instruction overrides.
- **Amber Signals**: Probable model drift or high risk scores (>70%).
- **Purple Signals**: Human-in-the-loop escalations.

## Constraints
- **Read-Only**: The dashboard provides a post-mortem view and does not allow modification of session data or governance states.
- **Org Isolation**: All queries are strictly scoped to the active `org_id`.
